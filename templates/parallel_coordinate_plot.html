Parallel Coordinate Plot

Great At:

  • Parallel coordinate plots are a powerful way to display multivariate data.
  • All axes are parallel to one another,
  • Each variable has its own line and is scaled from the minimum to the maximum data values.
  • Really effective, if plot is interactive.
  • Helpful in identifying clusters and variables with common patterns.
  • Great a displaying time series data for multiple variables.

    • Tips:

  • Normalize/Standardise
      • If variables vary in the scale of data, normalize or standardize the data before plotting.
  • Alpha-blending
      • PCP suffers from overplotting and alpha blending could be used to see overplotting issues in the graph.
      The lines are made slightly opaque on a scale of 0 to 1.
  • Spline
      • Multiple variables could have the same value, making it hard to see paths and patterns.
      Spline could be used to fix this, but will still suff from labeling issues.
  • Smoothers
      • Could be used to estimate true value as a weighted average of neighboring data.
      The bin width will be an issue when plotting discrete data; the data will be incorrectly binned.
  • Gaps
      • If gaps exist, add some data to smooth it out.
      Leave the gaps to show inconsistency in data.
  • Scale
      • PCP has built in ways to normalize and standardize the data.
  • std; robust, unimax, gloablminmax, center, centerObs
  • Missing
      • Ways to treat outliers in the data:
      Exclude it; Use mean, median or random.
  • Looking for features:
      • Outliers
      Patterns
      Gaps
      Clusters
      Separated Groups

    Code:

    library(ggplot2)
    library(GGally)
    df <- mtcars
    ggparcoord(df, columns = c(1:7), groupColumn = "cyl", alphaLines = .4)
