Histogram

Great At:

  • Shows count, relative frequency, cumulative frequency or density
  • Relative frequency or density is the count and total of each bin summed up to one.
  • Relative frequency: count/total for each bin, sum up to 1.
  • Boundaries may change the shape of the histogram.
  • Boundaries could be right close or right open:
      • left-open, right-open: (4, 8) 4 < x < 8
      left-closed, right-open: [4, 8) 4 ≤ x < 8
      left-open, right-closed: (4, 8] 4 < x ≤ 8
      left-closed, right-closed: (4, 8] 4 ≤ x ≤ 8
  • Unlike a barchart, histograms have no gaps between intervals.
  • Spotting distribution features

    • Tips:

  • Bins
      • Default is 30. Use binwidth to override.
      Always explore other bin widths and find the best one to showcase your data.
      Good general rule is to use the square root of the number of observations (round up the result).
  • Rounding
      • Varying the bin width will help identify rounding errors.
      Gaps in a histogram with small bin widths will indicate rounding patterns.
      Eg. if the shape of the histogram stays the same as bin width is varied, there's rounding errors.
  • Top Coding
      • At times, top coded data is in place to keep the anonymity of data.
      The last value has a “+” category to show that values still continue going up.
      There’s a large bin at the end to show that there’s a possibility of a lot more data.
      Top coding is a problem if there's a lot of data being hidden.
  • Continuous data
      • Why only use histograms for continuous data?
      The bin width will be an issue when plotting discrete data; the data will be incorrectly binned.
  • Looking for features:
      • Asymmetry
      Outliers
      Multimodality
      Gaps
      Heaping / Rounding
      Impossibilities / Errors

    Code:

    library("ggplot2")
    df <- mtcars
    ggplot(df, aes(mpg)) +
      geom_histogram(bins=14, color='white', fill='green')
